根据enum调用不同的方法

背景

根据enum的不同调用不同的方法

常见方法

  • if else

    if (enumValue.equals(ENUMVALUE1))
    method1(param);
else if (enumValue.equals(ENUMVALUE2))
    method2(param);
else
    // ...

  • switch case

    switch (enumValue) {
    case ENUMVALUE1:
        method1(param);
        break;
    case ENUMVALUE2:
        method2(param);
        break;
    default:
        // ...
        break;
}

更优解

在enum定义中定义方法，调用时直接调用enum内实现的方法，无需判断enum的值

public enum iEnum {
    ENUMVALUE1 {
        @Override
        public void method(int param) {
            // do something method1() will do...
        }
    }
    ENUMVALUE2 {
        @Override
        public void method(int param) {
            // do something method2() will do...
        }
    }
    public abstract void method(int param);
}

如此定义之后，在之前if else或switch case的地方直接调用:

enumValue.method(param);

将会根据enumValue的不同，调用不同enum内实现的method

Tips

我们也许习惯了根据不同String的值创建enum，比如

public enum iEnum {
    ENUMVALUE1("enumValue1"),
    ENUMVALUE2("enumValue2");
    
    private String name;
    private iEnum(String name) {
        this.name = name;
    }
}

想要将String的值继续保留且定义不同method，可以加一个return type为String的方法

public enum iEnum {
    ENUMVALUE1 {
        @Override
        public void method(int param) {
            // do something method1() will do...
        }
        
        @Override
        public String name() {
            return "enumValue1";
        }
    }
    ENUMVALUE2 {
        @Override
        public void method(int param) {
            // do something method2() will do...
        }
        
        @Override
        public String name() {
            return "enumValue2";
        }
    }
    
    public abstract void method(int param);
    public abstract String name();
}
